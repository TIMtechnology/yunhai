预测模型说明

本系统采用 规则引擎（fuzzy v2 + Archetype） 与 标注驱动 ML v6 的混合架构：日出窗口 03:00–07:00 由两者按点位权重融合给出云海概率，同时保留完整气象因子与底层观测。每个预报小时独立计算；综合场景标签（如「较可能日出云海」）由云海概率、日出概率与能见度证据联合判定。

三层因子展示：① 模型层（ML 概率、规则参考、融合权重）② 规则层（加权评分因子，驱动雷达图）③ 观测层（分层云量、垂直场、云海型态等，不参与加权，供专业解读）。

ML v6 · 日出窗口混合预测

项目说明
启用时段当地 03:00–07:00，且该观景点标注 ≥30 天且存在点位模型
算法Logistic 回归 + StandardScaler + 可选概率校准，class_weight=balanced
训练样本SQLite 标注库 cloudsea_labels（full / partial / none）
特征43 维日聚合：云量/能见度/RH850·700、逆温、Type A/B 小时计数、地形（海拔窗/填谷）、可观测场等；不含局地水体静态特征（水体判据仅走规则）
验证留一日交叉验证 LOOCV；五女山点将台 v6 增强特征 LOOCV ≈ 82%（baseline ≈ 73%）
融合权重五女山默认 ML 65%；其他内置景区 60%；近库水体雾或低能见度谷地雾时降至 40–50%，让规则 Type B 补偿主导
输出融合概率 + 规则因子 + ML 特征贡献 + 底层观测

训练：scripts/train_cloudsea_model.py · 点位模型：data/cloudsea/models/{spot}__{viewpoint}.pkl

Archetype · 金标准型态

基于有/无云海日对比与点位地形归纳，用于低云补偿、概率上下限与 plausibility cap：

型态特征规则处理
Type A高能见度（≥5 km）、模式低中云偏少、RH850 适中有效低云补偿；高能见度山谷填云
Type B低能见度层云/谷地雾；含水体蒸发型（近库 + 模式低能见度 + RH≥72%）层云/雾补偿；概率下限 ~48%；解除干廓线过度压分
Type C峰顶俯瞰（peak_overlook），谷地云海在日出扇区可观测可观测场 + 扇区气象；峰顶专属权重
雾型排除近地面饱和雾、中层云偏高且无观赏结构硬上限 ~25%，避免误判为观赏云海

五女山等点位可在景区 JSON 配置 local_water（如桓龙湖），供规则引擎识别近库晨雾，不进入 ML 特征向量。

因子权重理想条件
