预测模型说明

本系统采用 规则引擎（fuzzy v2 + Archetype） 与 标注驱动 ML v2 的混合架构：日出窗口 03:00–07:00 由 ML 给出云海概率，同时保留完整气象因子与底层观测数据。每个预报小时独立计算。

三层因子展示：① 模型层（ML 概率、规则参考、特征贡献）② 规则层（10 项加权评分因子，驱动雷达图）③ 观测层（分层云量、垂直场剖面、云海型态等，不参与加权，供专业解读）。

ML v2 · 日出窗口混合预测

项目说明
启用时段当地 03:00–07:00（日出前核心窗口）
算法Logistic 回归 + StandardScaler，class_weight=balanced
训练样本SQLite 标注库 cloudsea_labels（full/partial/none）
特征22 维日聚合：云量、能见度、RH850/700、逆温、Type A/B 小时计数等
验证留一日交叉验证 LOOCV（小样本下最可靠指标）
当前性能32 标注日 LOOCV ≈ 71.9%，AUC ≈ 0.749（优于纯规则 ~67%）
输出ML 概率 + 完整规则因子 + 底层观测（约 21 项）

训练脚本：scripts/train_cloudsea_model.py · 模型文件：data/cloudsea/models/cloudsea_ml_v2.pkl

Archetype · 五女山金标准型态

基于有/无云海日对比归纳，用于低云补偿与上限/下限约束：

型态特征规则处理
Type A高能见度（≥5 km）、低中云、RH850≤55有效低云补偿至 ~30%；概率下限 ~58%
Type B低能见度（≤500 m）、RH≤85、RH850≤45层云型补偿；概率下限 ~48%
雾型排除近地面饱和雾、中层云偏高硬上限 ~25%，避免误判为观赏云海
