系统架构

生产环境为单 Docker 镜像（linux/amd64）：FastAPI 托管 API + 静态前端 + Redis 缓存。预测编排由 predictor.py 统一完成，云海 ML 模型打包在镜像 /app/models/（独立于标注数据卷）。

预测流水线

  1. 按观景点 WGS84 坐标请求 Open-Meteo 5 天逐小时场（含气压层）。
  2. 逐小时调用 score_cloudsea 生成规则因子与参考概率。
  3. 03:00–07:00 且 ML 启用时，以日聚合特征调用 predict_day_cloudsea，合并 ML 概率与全部因子。
  4. 并行计算日出窗口评分与综合场景标签（晴日日出 / 云海日出等）。
  5. 当天已发生时段可选 Himawari 红外区域云量轻量校正。
