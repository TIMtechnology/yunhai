预测模型说明

本系统采用 规则引擎（fuzzy v2 + Archetype） 与 标注驱动 ML v7 的混合架构：日出窗口 03:00–07:00 由两者按点位权重融合给出云海概率，同时保留完整气象因子与底层观测。2026-06 起线上默认使用 v7 点位模型。详见 更新说明。

三层因子展示：① 模型层（ML 概率、规则参考、融合权重）② 规则层（加权评分因子，驱动雷达图）③ 观测层（分层云量、垂直场、云海型态等，不参与加权，供专业解读）。

v7 相对 v6 为何更准？

v6 仅聚合日出 03–07 点场；v7 额外利用前夜湿度与低云演变，对「预报仍湿、实况已干」类虚高 case 更敏感：

信号示例特征作用
傍晚段evening_rh_mean, evening_cloud_low_mean前夜水汽是否已经建立
夜间段night_rh_mean, night_inversion_mean夜间维持 / 消散趋势
跨段趋势delta_rh_night_to_dawn, rh_monotonic_nightnight→dawn 是否「预报升、实况降」

Archetype · 金标准型态

基于有/无云海日对比与点位地形归纳，用于低云补偿、概率上下限与 plausibility cap（规则层，与 ML 并行）：

型态特征规则处理
Type A高能见度（≥5 km）、模式低中云偏少、RH850 适中有效低云补偿；高能见度山谷填云
Type B低能见度层云/谷地雾；含水体蒸发型（近库 + 模式低能见度 + RH≥72%）层云/雾补偿；概率下限 ~48%；解除干廓线过度压分
Type C峰顶俯瞰（peak_overlook），谷地云海在日出扇区可观测可观测场 + 扇区气象；峰顶专属权重
雾型排除近地面饱和雾、中层云偏高且无观赏结构硬上限 ~25%，避免误判为观赏云海

近库水体判据走规则引擎，不进入 ML 特征向量。
